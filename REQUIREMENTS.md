# 图片描述生成工具 - 需求文档

## 项目概述

开发一个 GUI 工具，使用本地多模态 AI 模型（Llama GGUF 格式）对图片进行自动描述，
将图片基本信息和 AI 描述以 JSON 格式本地存储（一文件一图），方便后续用于生图 prompt。

---

## 功能需求

### 1. 图片选择
- 支持选择单张图片
- 支持选择目录（批量处理目录下所有图片）
- 支持的图片格式：JPG/JPEG、PNG、WEBP、BMP、GIF（静态）等常见格式

### 2. 图片基本信息采集
对每张图片自动采集以下信息：

| 字段 | 说明 |
|------|------|
| MD5 | 图片文件的 MD5 哈希值（用于去重/唯一标识） |
| 尺寸 | 宽×高度（像素） |
| 大小 | 文件大小（字节） |
| 类型 | 图片格式（如 JPEG、PNG） |
| 路径 | 原图片的绝对路径 |
| 处理时间 | AI 描述生成的时间戳 |

### 3. AI 描述生成
- 调用**llama-server 启动的多模态模型服务**（HTTP API）对图片进行描述
- 输出自然语言描述（prompt 友好型，适合后续用于生图）
- 支持配置服务地址、上下文长度、温度等生成参数

### 4. 标签管理
- **自动生成**：可由 AI 模型同时输出关键词标签
- **手动添加**：用户可后续补充/修改标签
- 标签支持多个，以数组形式存储

### 5. 数据存储
- **每个图片对应一个独立的 JSON 文件**
- JSON 文件与原图片同名，存放在指定输出目录（或原目录）
- 方便用户手动修改，也方便后续导入 SQLite

### 6. 批量处理
- 支持批量处理目录下的所有图片
- GUI 显示处理进度（进度条、当前处理图片、已处理数量）
- 支持断点续处理（已处理过的 MD5 跳过或覆盖，运行时可选）

### 7. GUI 界面功能
- 图片选择器（文件/目录）
- 服务配置（llama-server API 地址、上下文长度、温度等参数）
- 处理进度展示
- 处理结果列表（可预览、可手动编辑描述和标签）
- 去重策略选择（跳过/覆盖）
- 输出目录设置

---

## 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 批量处理时支持并发数配置（受显存/内存限制） |
| 隐私 | 所有数据本地存储，不上传云端 |
| 跨平台 | 支持 Windows/macOS/Linux |
| 易用性 | GUI 界面简洁直观，无需命令行操作 |

## 技术选型

| 模块 | 方案 |
|------|------|
| 开发语言 | Python |
| GUI 框架 | CustomTkinter（现代化外观，轻量） |

## JSON 文件结构

**文件名**: `photo.jpg.json`（与原图片同名，加 `.json` 后缀）

```json
{
  "md5": "abc123def456...",
  "source_path": "D:/images/photo.jpg",
  "width": 1920,
  "height": 1080,
  "size_bytes": 524288,
  "format": "JPEG",
  "description": "一张夕阳下的海滩照片，金色的阳光洒在海面上，波浪轻拍礁石，天空呈现橙红色渐变...",
  "tags": ["海滩", "夕阳", "风景", "自然"],
  "created_at": "2026-03-14T10:30:00",
  "updated_at": "2026-03-14T10:30:00"
}
```

## 使用流程

1. 打开 GUI 界面
2. 选择图片或目录
3. 设置输出目录（可选，默认与原目录相同）
4. 配置 llama-server API 地址和参数
5. 选择去重策略（跳过/覆盖）
6. 点击"开始处理"
7. 等待处理完成，可实时预览和编辑结果
8. 手动修改 JSON 文件（随时可编辑）
